> **Key finding:** XGBoost outperforms the baseline by ~20 R² points, confirming that the relationship between features and house price is non-linear. Linear models hit a ceiling that tree-based methods break through.

## 📌 Business Problem

Real estate platforms, mortgage lenders, and property investors need reliable home value estimates. This project builds a model that predicts the **median house value** of a California census block based on location, housing characteristics, and demographic data.

## 🔍 What This Project Covers

### 1 — Exploratory Data Analysis (Notebook 01)
- Missing value detection and treatment
- Target variable distribution analysis
- Correlation heatmap across all features
- Geographic visualisation (price by lat/lon)
- Ocean proximity analysis
- Feature engineering (3 new ratio features)

### 2 — Linear Regression Assumption Testing (Notebook 02)
Before modelling, the data is tested against the 5 formal assumptions of Linear Regression:

| Assumption | Test Used | Result |
|---|---|---|
| Linearity | Fitted vs Residuals plot | Partial |
| Independence | Durbin-Watson | Pass |
| Homoscedasticity | Breusch-Pagan | Fail (heteroscedastic) |
| Normality of Residuals | Q-Q Plot + Shapiro-Wilk | Approximate pass |
| No Multicollinearity | VIF (Variance Inflation Factor) | Moderate |

> Testing and acknowledging assumption violations is what makes this project rigorous — the model results are interpreted in context.

### 3 — Model Improvements (Notebook 03)
Six approaches are trained, evaluated, and compared:
- **Baseline** Linear Regression
- **Log-transformed** target variable
- **Ridge Regression** (L2 regularisation)
- **5-Fold Cross-Validation** for reliable performance estimates
- **XGBoost** (gradient boosted trees)
- **GridSearchCV** hyperparameter tuning on Ridge

## ⚙️ Feature Engineering

Three new features were created from existing columns:

| Feature | Formula | Rationale |
|---|---|---|
| `rooms_per_household` | total_rooms / households | Avg rooms per home, not per census block |
| `bedrooms_per_room` | total_bedrooms / total_rooms | Living space vs bedroom density |
| `population_per_household` | population / households | Proxy for average household size |

## 💡 Using the Prediction Function

The trained XGBoost model can be used directly to predict a house price:

```python
import joblib
import pandas as pd

model  = joblib.load("models/xgboost_model.pkl")
scaler = joblib.load("models/scaler.pkl")

sample_house = {
    "longitude": -122.23, "latitude": 37.88,
    "housing_median_age": 41.0, "total_rooms": 880.0,
    "total_bedrooms": 129.0, "population": 322.0,
    "households": 126.0, "median_income": 8.3252,
    "rooms_per_household": 6.984, "bedrooms_per_room": 0.147,
    "population_per_household": 2.556,
    "ocean_proximity_INLAND": 0, "ocean_proximity_ISLAND": 0,
    "ocean_proximity_NEAR BAY": 1, "ocean_proximity_NEAR OCEAN": 0
}

input_df     = pd.DataFrame([sample_house])
input_scaled = scaler.transform(input_df)
prediction   = model.predict(input_scaled)[0]

print(f"Predicted price: ${prediction:,.0f}")
```
